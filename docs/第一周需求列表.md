# 第一周需求列表

## 1. 第一周总目标

第一周目标是把 `Insight-front-kg` 做成可直接演示、可和后端 Agent 组对接的知识图谱前端工作台。

本周完成后，应能稳定跑通：

```text
mock insight_payload
-> KG 服务生成 graph_payload
-> 前端展示知识图谱
-> 可选 Neo4j 入库
-> 记录验证结果
```

当前验收口径：

- 可以先用 mock 数据跑通。
- 必须保留和后端真实 `insight_payload` 对接的接口。
- Neo4j 是增强能力，不是第一周前端展示的硬依赖。
- GraphRAG、CEO 决策问答、复杂图查询 API 不作为第一周完成项。

## 2. 需求状态说明

需求状态建议使用：

## 3. R1 mock 数据闭环

优先级：P0

目标：

```text
使用固定 mock insight_payload 验证图谱生成和前端展示闭环。
```

输入：

```text
samples/new-energy-insight-payload.json
frontend/public/samples/new-energy-insight-payload.json
```

输出：

```text
graph_payload
samples/new-energy-graph-payload.json
```

功能要求：

- 前端可以加载“新能源汽车”mock 数据。
- KG 服务可以根据 mock `insight_payload` 生成 `graph_payload`。
- mock 数据必须包含 `task_profile`、`chain_skeleton`、`segment_companies`、`company_cards`、`source_index`。
- mock 数据必须包含结构化 `relationship_clues`。
- mock 数据必须包含字段级 `field_evidence`。

验收标准：

- `graph_nodes.length > 0`
- `company_edges.length > 0`
- `profile_edges.length > 0`
- `evidence_chunks.length > 0`
- 前端点击“加载 mock”后能看到 JSON。
- 前端点击“生成图谱”后能看到产业关系图。

当前基线结果：

```text
graph_nodes=43
company_edges=2
profile_edges=18
evidence_chunks=27
```

## 4. R2 KG 服务接口

优先级：P0

目标：

```text
提供后端 Agent 组和前端都能调用的 REST 图谱服务。
```

服务地址：

```text
http://localhost:8008
```

接口要求：

```text
GET  /health
POST /api/graph/build
POST /api/graph/persist
POST /api/graph/build-and-persist
```

接口说明：

- `/health` 返回服务状态。
- `/api/graph/build` 输入 `insight_payload`，输出 `graph_payload`，不写 Neo4j。
- `/api/graph/persist` 输入 `graph_payload`，尝试写 Neo4j。
- `/api/graph/build-and-persist` 输入 `insight_payload`，生成图谱并尝试写 Neo4j。

验收标准：

- `/health` 返回 `status=ok`。
- `/api/graph/build` 能处理 mock `insight_payload`。
- `/api/graph/build` 返回的 `graph_payload` 包含 `graph_nodes`、`company_edges`、`profile_edges`、`evidence_chunks`。
- Neo4j 未配置时，接口不崩溃。
- Neo4j 未配置时，`persistence_meta.status` 可以是 `skipped`。

```powershell

python -m pytest -q
```

## 5. R3 前端图谱结果区

优先级：P0

目标：

```text
前端可以作为独立图谱工作台运行，并展示 graph_payload。
```

页面地址：

```text
http://localhost:3001
```

功能要求：

- 首页就是图谱工作台，不做宣传页。
- 支持加载 mock 数据。
- 支持粘贴后端 JSON。
- 支持调用 KG 服务生成图谱。
- 支持调用 KG 服务生成图谱并尝试入库。
- 显示图谱统计：节点数、公司边、画像边、入库状态。
- 展示产业链摘要。
- 展示公司关系图。
- 点击公司节点后展开画像关系。
- 点击关系边后展示关系名、方向、置信度、状态、证据文本和来源链接。
- 没有 `company_edges` 时显示清晰空状态。

输入：

```text

```

输出：

```text
前端图谱展示
```

验收标准：

- `npm run build` 通过。
- 打开 `http://localhost:3001` 返回 200。
- 点击“加载 mock”可以加载新能源汽车样例。
- 点击“生成图谱”可以看到公司关系图。
- 点击公司节点可以展开业务线、产品线、核心技术关系。
- 点击边可以看到证据来源。

## 6. R4 后端 Agent 组对接

优先级：P0

目标：

```text
明确后端 Agent 组需要交付的 insight_payload 字段，保证后续真实数据可以接入。
```

后端输入要求：

```text
task_profile.industry
chain_skeleton
segment_companies
company_cards
source_index
```

关键字段要求：

```text
company_cards[].relationship_clues
company_cards[].field_evidence
company_cards[].key_sources
company_cards[].evidence_excerpts
source_index[].source_grade
```

结构化 `relationship_clues` 示例：

```text
目标企业=比亚迪；关系层级=industry_template；关系名称=动力电池供应；方向=本企业->目标企业；证据=宁德时代为比亚迪提供动力电池相关产品。；来源=https://example.com/catl-byd；来源等级=A；解释=宁德时代向比亚迪提供动力电池相关产品
```

验收标准：

- 后端给出的 `insight_payload` 可以直接传给 `/api/graph/build`。
- 后端给出的 `relationship_clues` 能生成非空 `company_edges`。
- 后端给出的 `field_evidence` 能支撑 `profile_edges` 的证据展示。
- 后端给出的 `source_index` 可以支撑证据来源展示。

对接文档：

```text
docs/api-contract.md
```

## 7. R5 Neo4j 可选入库

优先级：P1

目标：

```text
图谱可以在 Neo4j 可用时入库；Neo4j 不可用时不阻塞前端展示。
```

Neo4j 启动命令：

```powershell
cd D:\AAA_Favio_2026\AI_exploring\DigitalChina\Insight-front-kg
docker compose -f docker-compose.neo4j.yml up -d
```

环境变量：

```text
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your_password
GRAPH_AUTO_PERSIST=true
```

验收标准：

- Neo4j 未配置时，前端仍可生成和展示图谱。
- Neo4j 未配置时，`persistence_meta.status=skipped`。
- Neo4j 配置成功时，`persistence_meta.status=persisted`。
- Neo4j 连接失败时，`persistence_meta.status=failed`，并记录失败原因。

第一周说明：

```text
Neo4j 入库是增强项，不是前端图谱展示的硬依赖。
```

## 8. R6 第一周演示路径

优先级：P0

目标：

```text
形成一个可复现的第一周演示流程。
```

演示步骤：

```text
1. 启动 KG 服务。
2. 启动前端。
3. 打开 http://localhost:3001。
4. 点击“加载 mock”。
5. 点击“生成图谱”。
6. 展示节点数、公司边、画像边、证据片段数量。
7. 点击公司节点，展示画像关系。
8. 点击关系边，展示证据文本和来源链接。
9. 说明 Neo4j 当前入库状态。
10. 打开 docs/api-contract.md，说明后端 Agent 组需要输出哪些字段。
```

验收标准：

- 演示全程不依赖真实搜索。
- 演示全程可以解释 `insight_payload -> graph_payload -> 前端图谱`。
- 能说明后端真实数据接入后替换的是 `insight_payload`。
- 能说明 Neo4j 是可选入库，不影响第一周图谱展示。

第一周任务可以判定完成，当且仅当：

- R1 mock 数据闭环完成。

